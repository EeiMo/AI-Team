## 四、问题诊断：SSH 连接失败根因

生产服务器的 SSH 配置仅允许：
1. **公钥认证** — 本机公钥未添加到 `~/.ssh/authorized_keys`
2. **密码认证** — 密码未存储

**建议修复**：

```bash
# 方法一：添加本机公钥到服务器（需密码）
ssh-copy-id -i ~/.ssh/id_ed25519.pub eeimoo@eeimoo.cn

# 方法二：或使用以下命令一次性添加（需密码）
cat ~/.ssh/id_ed25519.pub | ssh eeimoo@eeimoo.cn "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"
```
