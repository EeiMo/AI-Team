## 五、修复记录

### 修复项 1：Docker Socket 权限
- 通过 `sudo chmod 666 /var/run/docker.sock` 修复

### 修复项 2：Snap Docker compose 二进制不兼容
- Snap docker 的 wrapper 脚本有 confinement 限制
- 直接调用 `/snap/docker/3505/usr/libexec/docker/cli-plugins/docker-compose` 绕过

### 修复项 3：后端构建失败（better-sqlite3 需编译）
- app.Dockerfile builder stage 添加 `apk add --no-cache python3 build-base`

### 修复项 4：Mock 文件导致 TypeScript 编译失败
- tsconfig.json `exclude` 中添加 `src/__mocks__`

### 修复项 5：前端 lockfile 不一致
- nginx.Dockerfile 中 `npm ci` 改为 `npm install`

### 修复项 6：GlobalThis 类型错误
- `test-setup.ts` 中 `globalThis.IS_TEST` 改为 `(globalThis as any).IS_TEST`

### 修复项 7：JWT_SECRET 未设置
- `.env.staging` 添加 `JWT_SECRET=staging_jwt_secret_dev_only_2026`
- `docker-compose.staging.yml` app 环境变量添加 `JWT_SECRET`

### 修复项 8：DATABASE_URL 中 PG_PASSWORD 变量未解析
- `.env.staging` 中 `DATABASE_URL` 硬编码密码，移除变量引用 `{PG_PASSWORD}`

### 修复项 9：PG_PASSWORD 在 Docker 全量重建后未正确初始化
- 初始 pg 容器启动时密码为空（`rolpassword` 为空）
- 通过 `ALTER USER vote_user WITH PASSWORD 'vote_dev_pass'` 修复

### 修复项 10：缺少 .dockerignore 导致构建上下文过大被 OOM kill
- 创建 `.dockerignore`，排除 `node_modules/`、`.git/`等
