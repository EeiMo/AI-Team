## 三、数据泄露风险验证

### 3.1 敏感字段存储检查

| 检查项 | 结果 |
|--------|------|
| 数据库是否存密码/token | ❌ 无 — users 表仅 `user_id, team_id, display_name, avatar_url` |
| votes 表是否有 PII | ❌ 无 — `creator_id` 飞书 open_id，非手机/邮箱 |
| audit_logs 表是否有敏感字段 | ❌ 无 — 仅记录 title + status |
| 日志中是否含 token/secret | ✅ 已检查 — 无泄露 |

### 3.2 匿名投票脱敏

| 检查项 | 结果 |
|--------|------|
| 匿名模式 voters 字段 | ✅ 始终返回空（已验证 v2 行为） |
| 审计日志不含 user_votes 详情 | ✅ 仅记录 DELETE_VOTE 操作级别日志 |

### 3.3 已删投票详情访问

- 创建者可通过 `GET /api/votes/:id` 查看已删投票详情（含选项、候选人）
- 非创建者也可查看（若知晓 UUID）— **PEN-002 残余风险**
  - 风险等级：P2（信息泄露，非数据修改）
  - 缓解：UUID v7 不可枚举、已删投票不出现于列表
  - 已转迭代 backlog

## 四、R-003 (audit_logs UUID 类型) 验证

| 检查项 | DDL 声明 | 实际 | 判定 |
|--------|----------|------|------|
| `audit_logs.user_id` | `VARCHAR(64)` | `character varying(64)` | ✅ 与 DDL 一致 |
| `audit_logs.team_id` | `VARCHAR(64)` | `character varying(64)` | ✅ 与 DDL 一致 |
| `votes.deleted_by` | `VARCHAR(64)` | `character varying(64)` | ✅ 与 DDL 一致 |

**说明**：R-003 在 v3 Go/No-Go 中已裁定为「与 ACHe v3 文档一致，驳回阻断」—— DDL 使用 VARCHAR(64) 存储飞书 `user_id` 原始值（如 `ou_xxx`），不转换为 `users.id` UUID。这是业务设计选择，非安全缺陷。
