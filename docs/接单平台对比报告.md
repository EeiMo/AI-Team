## 一、平台独立分析

### 1. 猪八戒网（zbj.com）

| 维度 | 说明 |
|------|------|
| **平台简介** | 国内最大的企业服务交易平台，成立于2006年，覆盖设计、开发、营销、工商财税等品类，月活雇主超千万。 |
| **项目类型** | 网站/小程序/APP开发、UI设计、营销策划、商标注册等，技术类以中小型为主（1-20万）。 |
| **入驻门槛** | 免费注册开店，但需实名认证和资质审核。流量竞争激烈，付费投标/会员制获取流量。 |
| **抽成比例** | 平台抽佣约5%-20%（视会员等级和项目类型）。免费用户抽成高，付费会员（年费数千至数万）可降低。 |
| **支付保障** | 平台托管资金，验收后付尾款。但纠纷处理偏向雇主方。 |
| **适合团队规模** | 3-10人团队，有完整交付能力。 |
| **优点** | 流量大、需求多、品类全；有资金托管保障；平台知名度高，容易建立信任。 |
| **缺点** | 竞争极度激烈，报价内卷严重；平台抽成偏高；低质量需求占比大，筛选成本高；需要持续投入会员费才能获取优质标。 |

> **对我们团队的建议**：适合用来刷案例和经验，但不适合作为主要收入来源。可通过猪八戒定位中高端项目（5-20万区间），与低端工作室错位竞争。

### 2. 程序员客栈（proginn.com）

| 维度 | 说明 |
|------|------|
| **平台简介** | 程序员自由工作平台，主打"云端开发团队"模式，注册开发者123万+，定位BAT级别技术人才。企业发布需求，平台匹配开发者。 |
| **项目类型** | 整包开发（小程序/Web/APP）、技术咨询、项目驻场、远程兼职。项目金额5-50万不等。 |
| **入驻门槛** | 开发者需实名认证+技术等级评估（学历/经验/项目经历）。个人或工作室均可入驻，通过审核后参与竞标或平台派单。 |
| **抽成比例** | 平台抽佣约10%-20%。企业会员有不同套餐。 |
| **支付保障** | 平台托管费用，分阶段验收后结算。先面试后开发，先验收再结算。 |
| **适合团队规模** | 2-8人团队，有完整技术栈和交付能力。 |
| **优点** | 开发者质量较高，BAT背景开发者占比大；需求相对精准，非纯比价模式；有项目管理工具配合交付；有"远程工作"趋势契合。 |
| **缺点** | 平台活跃项目数量不如猪八戒；抽成不低；项目接洽流程较长（面试→匹配→签约→开发）。 |

> **对我们团队的建议**：★ Top 推荐之一。定位匹配 —— 我们团队有Go+React+CI/CD完整能力，正好切中平台"高端技术团队"定位。包装为"全栈敏捷交付团队"具有竞争力。

## 三、Top 3 推荐平台及理由

### 🥇 第1名：程序员客栈（proginn.com）

**核心理由**：
1. **定位最匹配** —— 平台主打"BAT级别技术团队"，而我们正是Go+React全栈团队，有架构师、测试、运维全能力，完美切合平台高端定位。
2. **项目质量高** —— 项目金额5-50万区间，客单价高，不需与低端工作室打价格战。
3. **入驻门槛成优势** —— 审核制筛掉了很多低质量竞争者，我们8人完整建制的团队在这个平台属于稀缺资源。
4. **支付保障好** —— 分阶段验收付款，配合我们的CI/CD自动化交付能力，能让客户看到实打实的进度。

### 🥈 第2名：电鸭社区（eleduck.com）

**核心理由**：
1. **零抽成** —— 100%收入归团队，无平台隐形成本。
2. **高质量技术社区** —— 社区氛围好，客户多为懂技术的创业团队或企业技术负责人，沟通成本低。
3. **远程趋势契合** —— 团队的飞书+CI/CD远程协作能力，在远程工作场景中是一大卖点。
4. **口碑积累价值** —— 在电鸭建立团队品牌后，会持续带来主动咨询的需求，形成复利效应。

### 🥉 第3名：猪八戒网（zbj.com）— 仅限中高端定位

**核心理由**：
1. **流量入口** —— 作为补充渠道获取项目曝光，不能完全放弃这个最大流量池。
2. **差异化竞争** —— 团队不接1-3万的"小单"，专注5-20万的中大型项目，利用完整团队建制和CI/CD能力与低端工作室拉开差距。
3. **新品打磨** —— 猪八戒的低端需求虽然多，但可以用来自我训练交付流程和建立案例库。

> ⚠️ **不建议**：在一品威客、码市、开发邦投入大量时间。要么流量不够、要么项目质量不匹配。仅作为观察窗口注册即可。

## 四、建议起步策略

### 4.1 第一单怎么打

**总体原则**：宁丢不抢，选客户比选平台重要。前3单注重"做漂亮"而不是"赚大钱"。

**具体路径**：

```
第1步（第1周）：包装团队
  ├── 统一团队名称（建议："XXX Studio" 或 "蜻蜓全栈"）
  ├── 准备团队介绍页（含成员头像、技术栈、过往案例、交付流程）
  ├── 准备一个 Demo 项目（用现有的投票工具项目作为案例展示）
  └── 在程序员客栈 + 电鸭社区完成入驻

第2步（第2-3周）：试水小项目
  ├── 在程序员客栈投标 1-2 个 3-8 万的 Web 开发项目
  ├── 报价策略：比市场价低 10%-15%（前3单不追求利润最大化）
  ├── 目标：拿好评、拿案例、拿流程经验
  └── 用飞书搭建客户沟通SOP（需求确认→原型→开发→测试→部署）

第3步（第4-8周）：打磨交付SOP
  ├── 每次交付附带：代码仓库权限+API文档+部署文档+CI/CD流水线截图
  ├── 在程序员客栈积累5星评价
  └── 每完成一单，更新团队案例库
```

### 4.2 报价策略

| 项目类型 | 参考报价 | 策略说明 |
|---------|---------|---------|
| 小程序开发（标准功能） | 3-8万 | 模板化，用组件库提升复用率 |
| Web管理系统（中台） | 8-20万 | 按功能模块报价，突出CI/CD快速交付 |
| API/后端集成开发 | 5-15万 | 按接口数量+复杂度报价，Go性能优势是卖点 |
| 全栈项目（Web+小程序+管理后台） | 15-40万 | 按阶段报价，分3-4期付款 |
| 技术咨询/代码审查 | 3000-8000元/次 | 作为轻度接单和拓展客户关系的手段 |

**报价技巧**：
- 永远不要报一口价，拆分为"基础版+可选功能"结构
- 运维部署+CI/CD配置作为增值服务单独报价（额外+15%-20%）
- 明确售后维护范围（1个月免费bug修复，后续按维护合同算）

### 4.3 包装建议

**团队定位**：**"全栈敏捷交付团队 —— 从原型到部署，7天交付可运行版本"**

**核心卖点词**：
- 「全套CI/CD自动化部署」
- 「Go高并发后端 + React现代前端」
- 「飞书SSO即插即用」
- 「测试覆盖 + 安全审计」

**案例包装**：
- 将现有的投票工具项目整理为公开案例（需脱敏），包括：
  - 技术架构图
  - 功能清单
  - 部署效果截图
  - 开发周期（3天 MVP + 2天测试部署）
- 写 2-3 篇技术博客（Go后端实践 / React最佳实践 / CI/CD落地经验）发在掘金/知乎，作为团队技术能力的佐证

**差异化竞争点**（和同平台的其他工作室/开发者不同之处）：
1. **流水线交付** —— 不是做完了发压缩包，而是直接把CI/CD流水线敞开给客户看
2. **有测试有安全** —— 多数小团队没有测试和安全角色，我们有
3. **完整的项目经理制** —— 云起作为PM全程跟进需求，减少开发者和客户的直接摩擦
