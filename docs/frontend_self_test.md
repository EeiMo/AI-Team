## 六、已知问题 / 待确认

1. **删除后详情页刷新策略**：当前删除成功后在详情页使用 `window.location.reload()` 让 API 返回 `deleted: true`。如后端 404 处理未覆盖，需后端配合返回 200 + `deleted: true`。
2. **WS 重连后房间清理**：删除后服务端 `io.in().socketsLeave()` 清理房间，前端无需额外操作。
3. **移动端 320px 触控**：确认所有按钮 min-height/min-width 为 44px（WCAG 标准），CSS Modules 中已落实。
